# Terraform Skeleton

This folder is intentionally lightweight for the MVP phase.

## Purpose

Prepare the repository for a future Azure-first deployment path without introducing infrastructure complexity too early.

## Suggested next steps

1. Add a resource group, static hosting or container hosting target, and monitoring basics.
2. Introduce environment-specific variable files.
3. Add remote state once the deployment workflow is agreed.
4. Extend toward AKS, Key Vault, and GitOps only when the product truly needs them.
